Newsfilter project goals

In the last few years, Usenet deteriorated to the degree that makes content filtering a screaming necessity, rather than a nice improvement.

Thus the goal of this open development project, in its first, short-term stage is to turn Usenet back into a meaningful communication medium, and help people find most interesting information and correspondents.

In the future, this service together with other systems utilizing standardized semantic descriptions of entities and their relations should make the Net into an ecology of active software agents and a playground for distributed symbolic AI.
The latest information on project plan details (they constantly grow) see at :

Newsfilter project structure

The major parts would include:

  • Data storage facilities. The purpose of these is to accept and serve standardized descriptions of users, items, and their relations.
  • Filtering/recommendation software. Code that will take into account information about current user, and information about a given set of items (e.g., current messages in a newsgroup), and predict user interest to these items.
  • User interface. The interface should be a simple extension to existing Usenet access methods, such as Netscape and DejaNews. It should be able to mask/delete items (messages) the user is not likely to be interested in, reorder others in order of expected preference, and accept user feedback.
